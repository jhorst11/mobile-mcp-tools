# Native Android Requirements

To develop a native Android app, meet the Salesforce and Android requirements.

## Salesforce Requirements

- Salesforce Mobile SDK 13.0 or later for Android.
- A Salesforce [Developer Edition organization](dev-de.md) with a [connected app](connected-apps-howto.md).

The `SalesforceSDK` project is built with the Android 9 Pie (API 28) library.

## Android Requirements

To install the Android development environment, go to the [Mobile SDK Trailhead project](https://trailhead.salesforce.com/en/content/learn/projects/mobilesdk_setup_dev_tools?trail_id=mobile_sdk_intro) and do the following steps:

1.  [Install Common Components](https://trailhead.salesforce.com/en/content/learn/projects/mobilesdk_setup_dev_tools/mobilesdk_setup_common)
2.  [Set Up Your Android Development Environment](https://trailhead.salesforce.com/content/learn/projects/mobilesdk_setup_dev_tools/mobilesdk_setup_android)

Here’s a list of the items you install.

- Java JDK 17 or later—[www.oracle.com/downloads](http://www.oracle.com/downloads/).
- Latest version of Android Studio —[developer.android.com/sdk](http://developer.android.com/sdk/).
- Android SDK, including all APIs in the following range:
  - Minimum API: Android 9 Pie (API 28)
  - Target API: Android 15 (API 35)
- Android SDK Tools
- Android Virtual Device (AVD)

:::tip

- For best results, install all Android SDK versions recommended by the Android SDK Manager, and all available versions of Android SDK tools.
- On Windows, be sure to run Android Studio as administrator.

:::
